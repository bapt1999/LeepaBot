User Profile and Objectives
The background is a personal engineering journey driven by curiosity for AI behavioral architecture, context routing, and stateless cognitive pipelines. The primary goal is to build a highly conversational, context-aware, autonomous Discord chatbot that actively participates in discussions, shitposts, and organically reacts to the channel vibe without needing direct command prompts.

Core Architectural Features
The host uses the Python discord library, deployed 24/7 on a Google Cloud Platform e2-micro Linux server using a systemd background daemon. The external AI engine utilizes a multi-provider multiplexer supporting Gemini, Groq, DeepSeek, and OpenRouter. Sensitive data is secured in an environment file. The logic layer employs a multi-stage parallel-track probability filter to route messages based on engagement levels and conversational classifications. Prompts and data are decoupled from routing logic to ensure clean separation of concerns.

Phase 1: Foundation and Interface
The bot establishes a reliable Discord connection, separating event handling from core business logic. This phase ensures the basic skeleton can send and receive messages while maintaining thread safety for asynchronous operations.

Phase 2: Social Cognition and Routing
The bot utilizes a dynamic multi-state router. The engagement track handles Direct, Quoted, and Ambient mentions. The classification track flags content as Standard, Shitpost, Yelling, Wall of Text, or Quick Banter. These tags are used to dynamically assemble instructions for the LLM brain, allowing it to adapt its tone to the current channel energy.

Phase 3: Stateless Short-Term Memory
The bot maintains isolated Short-Term Memory queues for unique channel IDs. When the queue reaches capacity, a non-blocking background thread compresses logs into a dense running summary. Long-Term Memory extraction and RAG pipelines have been deliberately severed to prevent toxic trait compounding and context window hijacking. The bot now operates statelessly across sessions, relying purely on sharp, immediate contextual awareness and a resilient baseline persona rather than historical databases.

Phase 4: Behavioral Expression and Persona
The bot utilizes a strict Chain-of-Thought Cognitive Pipeline. An internal thinking block allows the LLM to process intent, anchor its emotional baseline, and plan its comedic angle before rendering dialogue. The persona is engineered as a highly enthusiastic, zero-ego, partner-in-crime fox-girl who acts as an enabler and peer to the users. A hard syntax limitation strictly forbids the use of question marks to ensure all responses are definitive, declarative statements. N-shot behavioral examples and the core persona are modularized in a dedicated prompts file. Physical Discord emoji reactions are decoupled from text generation and utilize a centralized, global inventory of custom emojis available across all servers. An Interception Matrix and Infinite Loop Slayer prevent dual-bot spam.

Phase 5: Thermal Tuning and Native Integration
The bot employs a centralized Thermal Tuning Matrix that maps context-specific temperature ranges to a normalized scale. A provider-agnostic translation layer scales these temperatures to model-specific limits. The Gemini integration utilizes a native REST endpoint to unlock thinkingConfig at a HIGH level. Frequency and presence penalties have been intentionally excluded to maintain compatibility with bleeding-edge preview models, relying entirely on structural prompt entropy to prevent repetition loops.

Phase 6: Infrastructure and DevOps
The system is deployed on a GCP Ubuntu instance managed by systemd. Code updates are synchronized via GitHub. For real-time monitoring and debugging, the logic layer supports raw JSON output logging directly to the terminal, allowing for live analysis of the bot's thinking blocks and internal mood states without needing to parse system journals.

Phase 7: Image Analysis
The bot is able to see images in jpg, jpeg and png types, and integrate their contents in its answers.

No new integrations are currently planned. The project is complete.
